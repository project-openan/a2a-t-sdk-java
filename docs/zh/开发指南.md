# Python SDK 开发指南

## 特性介绍

`a2a-t-sdk` 提供 A2A-T 协议在 Python 侧的客户端和服务端基础能力。SDK 将电信领域任务表达拆分为提示词资源、场景识别、槽位提取、槽位校验、任务提示词渲染和协商运行时，便于 Agent 开发者复用统一交互格式。

主要功能包括：

- 客户端生成 processed task prompt。
- 服务端校验 processed task prompt。
- 客户端和服务端均支持协商发起、接收和继续。
- 基于 `.env` 的统一运行时配置。
- 本地提示词资源加载和 OpenAI-compatible LLM 适配。

## 约束与限制

1. SDK 当前版本为 `0.1.8`，处于 Alpha 阶段。
2. Python 版本要求为 3.12+。
3. 内置资源覆盖有限，当前随包资源以 `subscribe_incident` 等样例场景为主。
4. 协商状态存储仅提供 `in_memory`，进程退出后状态丢失。
5. SDK 不提供 Agent HTTP 服务框架、注册中心客户端、鉴权和密钥管理能力，需由业务系统集成。

## 环境准备

### 1. 获取源码

```bash
git clone git@gitcode.com:OpenAN/a2a-t-sdk.git
cd a2a-t-sdk
```

### 2. 安装开发依赖

```bash
uv sync --dev
```

### 3. 准备配置

```bash
cp package_data/env.example package_data/.env
```

配置示例：

```properties
A2AT_LANGUAGE=zh-CN
A2AT_PROMPT_SOURCE_TYPE=local_file
A2AT_PROMPT_RESOURCE_LOCAL_ROOT_DIR=
```

```bash
uv run pytest
uv run ruff check .
uv run mypy src
```

## 核心 API

### 客户端门面

```python
from pathlib import Path

from a2a_t.client.a2at_client import A2ATClient

client = A2ATClient(env_path=Path("package_data/.env"))
result = client.generate_task_prompt("订阅 critical 级别的 ETH-LOS 故障通知")
```

`generate_task_prompt` 支持字符串和结构化字典输入，返回 `PromptGenerationResult`：

| 字段 | 说明 |
| --- | --- |
| `success` | 是否生成成功 |
| `prompt_text` | 成功时的 processed task prompt |
| `failure` | 失败时的标准错误信息 |

### 服务端门面

```python
from pathlib import Path

from a2a_t.server.a2at_server import A2ATServer

server = A2ATServer(env_path=Path("package_data/.env"))
result = server.check_task_prompt(processed_prompt_text=prompt_text)
```

`check_task_prompt` 返回字典：

| 字段 | 说明 |
| --- | --- |
| `success` | 是否校验通过 |
| `failure` | 失败时包含 `code`、`message`、`stage` |

### 协商接口

客户端和服务端门面均提供：

| 方法 | 说明 |
| --- | --- |
| `start_negotiation(input)` | 发起一轮协商 |
| `receive_negotiation(message, context)` | 接收并解析对端协商消息 |
| `continue_negotiation(input)` | 基于已有上下文继续协商 |

协商类型定义在 `a2a_t.negotiation.common.enums.NegotiationType`：

- `information`
- `clarification`
- `feasibility`
- `fulfillment`

## 集成开发流程

### 1. 客户端 Agent

客户端 Agent 通常按以下流程集成：

1. 接收用户自然语言或结构化输入。
2. 调用 `A2ATClient.generate_task_prompt` 生成 processed task prompt。
3. 将生成结果放入 A2A 消息正文或扩展字段。
4. 发送给目标 Agent。
5. 如果收到协商上下文，调用 `receive_negotiation` 和 `continue_negotiation` 生成下一轮消息。

示例：

```python
from pathlib import Path

from a2a_t.client.a2at_client import A2ATClient
from a2a_t.negotiation.common.enums import NegotiationStatus
from a2a_t.negotiation.common.models import ContinueNegotiationInput, NegotiationContext

client = A2ATClient(env_path=Path(".env"))

prompt_result = client.generate_task_prompt(
    {
        "scenario": "subscribe_incident",
        "objective": "Subscribe to incident notifications from network devices.",
    }
)

if not prompt_result.success:
    raise RuntimeError(prompt_result.failure.to_dict())

processed_prompt = prompt_result.prompt_text

# 收到服务端协商消息后继续协商
receive_result = client.receive_negotiation(server_message, server_context)
continue_payload = client.continue_negotiation(
    ContinueNegotiationInput(
        context=NegotiationContext.from_context(receive_result["context"]),
        status=NegotiationStatus.IN_PROGRESS,
        content_text=processed_prompt,
    )
)
```

### 2. 服务端 Agent

服务端 Agent 通常按以下流程集成：

1. 从 A2A 请求中提取 processed task prompt。
2. 调用 `A2ATServer.check_task_prompt`。
3. 校验通过后进入业务执行。
4. 校验失败或业务层发现信息不足时，调用 `start_negotiation` 或 `continue_negotiation`。
5. 将协商文本和上下文返回给客户端。

示例：

```python
from pathlib import Path

from a2a_t.negotiation.common.enums import NegotiationType
from a2a_t.negotiation.common.models import StartNegotiationInput
from a2a_t.server.a2at_server import A2ATServer

server = A2ATServer(env_path=Path(".env"))
result = server.check_task_prompt(processed_prompt_text=processed_prompt)

if result["success"]:
    business_result = execute_business(processed_prompt)
else:
    negotiation_payload = server.start_negotiation(
        StartNegotiationInput(
            type=NegotiationType.INFORMATION,
            content_text="Please provide missing subscription information.",
            facts={"failure": result["failure"]},
        )
    )
```

## 提示词资源扩展

自定义场景时，需要准备与内置资源结构一致的文件：

```text
prompt_resources/
  scenarios/zh-CN/scenarios.json
  slots/{scenario_code}/zh-CN/slot.json
  templates/{scenario_code}/zh-CN/template.md
  prompts/scenario_recognition/zh-CN/system.md
  prompts/scenario_recognition/zh-CN/user.md
  prompts/slot_extraction/zh-CN/system.md
  prompts/slot_extraction/zh-CN/user.md
  prompts/semantic_validation/zh-CN/system.md
  prompts/semantic_validation/zh-CN/user.md
```

然后在 `.env` 中配置：

```properties
A2AT_PROMPT_SOURCE_TYPE=local_file
A2AT_PROMPT_RESOURCE_LOCAL_ROOT_DIR={your_prompt_resources_root}
A2AT_LANGUAGE=zh-CN
```

建议新增资源后补充以下测试：

1. 场景识别测试。
2. 槽位提取测试。
3. 槽位 JSON Schema 校验测试。
4. 客户端生成和服务端校验的端到端测试。

## 示例工程

`a2a-t-samples` 提供两个 Python 示例：

| 示例 | 说明 |
| --- | --- |
| `samples/helloworld` | 最小双端流程，客户端生成 prompt，服务端校验后结束 |
| `samples/subscribe_incident` | incident 订阅流程，包含多轮信息补充协商 |

建议开发者先运行 `helloworld` 验证环境，再参考 `subscribe_incident` 集成业务层协商逻辑。
